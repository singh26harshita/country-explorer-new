# Take-Home Assignment

## Country Explorer

Thank you for your interest in the Frontend Engineer role at IBFD. As part of the process, we'd like you to dig into **Country Explorer**, an Angular 19 application that displays information about countries worldwide using the [REST Countries API](https://restcountries.com/).

The application was built as a quick prototype. It works, but the code quality needs improvement. The goal is to **review, refactor, and extend** it.

## Time and Scope

Please keep your effort to under 3 hours and submit within 7 days. You're free to prioritize as you see fit. If there's more you'd have done with extra time, let us know in your `SUBMISSION.md`.

## What to do

The following parts are listed in suggested order, but you decide how to spend your time. We expect production-quality code.

You're free to add libraries if needed, but please justify the choice in your `SUBMISSION.md`.

### Part 1: Code Review and Refactor

Review the existing codebase. In your `SUBMISSION.md` file, document the **top issues** you'd flag in a code review: what's wrong, why it matters, and how you'd fix it.

Then, improve the code based on your findings.

### Part 2: Build the Country Comparison Page

Build a new **Compare** page where users can select 2-3 countries and view them side by side.

Requirements:
- New route: `/compare`
- Country selector (users pick which countries to compare)
- Side-by-side display of key data: population, area, capital, region, currencies, languages, timezones, borders

Feel free to keep the design simple.

### Part 3 (Bonus): Performance

The country list re-renders inefficiently. If you still have time, optimize it.

Briefly explain what you optimized and why in your `SUBMISSION.md`.

1. **Code Review**: The top issues you found in the existing code and why they matter
2. **What you did**: Summary of changes you made and decisions you took
3. **What you'd do with more time**: What you'd prioritize next
4. **Tradeoffs**: Any tradeoffs you made and why
5. **How to run**: Any additional setup steps needed (if any)

There's no checklist of requirements that you pass or fail. We're looking at how you read and improve existing code, how you write and structure your own, and how you communicate your thinking.
